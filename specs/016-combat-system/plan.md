# Implementation Plan: Combat System тАФ Mana Threshold Auto-Battle

**Branch**: `016-combat-system` | **Date**: 2026-03-13 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/016-combat-system/spec.md`

## Summary

Replace the existing batch-computed combat loop with a server-authoritative, real-time turn-by-turn system. Each turn resolves on the server and is broadcast over WebSocket as it happens. Players accumulate mana through attacking and being hit, which fuels 3 auto-ability slots that fire automatically at threshold, plus 1 active slot with a per-turn timer window. A new Loadouts tab lets players configure their ability slots. A Pokemon-style full-screen combat overlay replaces the existing `CombatModal`. The admin panel gains an Abilities section for tuning ability properties.

## Technical Context

**Project Type**: Multiplayer web game (WebSocket-centric)
**Performance Goals**: Turn resolution тЙд 500ms server-side; active window timer accurate to ┬▒200ms
**Constraints**: Combat session state is in-memory only (Phase A); no Redis or external state store required
**Scale/Scope**: 1 active combat session per character; session manager is a singleton per backend process

### Key Design Details

#### Combat Engine (`combat-engine.ts`)

Pure functions тАФ no I/O, no timers. Takes a `CombatSession` snapshot and returns a `TurnResult`:

```
resolvePlaerTurn(session) тЖТ { events[], newState }
resolveEnemyTurn(session) тЖТ { events[], newState }
```

Turn order per spec FR-004:
1. `+= manaRegen`
2. Player auto-attack тЖТ roll dodge (enemy) тЖТ roll crit тЖТ apply damage тЖТ `+= manaOnHit`
3. Auto-ability resolution: sorted by priority desc тЖТ for each: `mana >= cost && offCooldown` тЖТ fire тЖТ deduct mana тЖТ start cooldown
4. Active window (returns тАФ caller manages timer)
5. Tick active effects (DoT damage, decrement durations, expire at 0)
6. Enemy turn: enemy auto-attack тЖТ roll player dodge тЖТ apply damage (reduced by defence) тЖТ `+= manaOnDamageTaken`

#### CombatSession (`combat-session.ts`)

Orchestrates the turn loop:
1. Call `combat-engine.resolvePlaerTurn()`
2. Send `combat:turn_result` (player phase)
3. Send `combat:active_window`
4. Set `setTimeout(TURN_TIMER_MS, onWindowExpired)`
5. On `trigger_active` тАФ cancel timer, apply ability, call `onWindowExpired`
6. `onWindowExpired()` тАФ call `combat-engine.resolveEnemyTurn()`
7. Send `combat:turn_result` (enemy phase)
8. Check HP тЖТ if ended: call `resolveWinRewards()` / `resolveLossOutcome()`, send `combat:end`
9. Else: start next turn from step 1

#### LoadoutPanel (`frontend/src/ui/LoadoutPanel.ts`)

Four slot cards arranged in a grid (3 auto + 1 active). Below: scrollable owned-ability list. Click an ability тЖТ assign to selected slot. Click an occupied slot тЖТ deselect (clear). Priority input per auto slot (number input). All interactions send `loadout:update` over WS. When `in_combat`, all inputs are `disabled` with a banner message.
